# Security Policy

## Supported Code

Security fixes are applied to the current active codebase in the default
branch and to the current production deployment workflow.

## Reporting a Vulnerability

Do not open public issues for suspected vulnerabilities.
Send reports to `office@wpowertech.ro`.

Include:

- A short description of the issue
- Affected routes, files, or components
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation, if available

## Response Process

1. The report is reviewed and triaged.
2. Reproduction is confirmed in the current codebase.
3. A fix is prepared and validated.
4. Disclosure timing is coordinated after remediation.

## Scope Examples

- Authentication or authorization bypass
- Header or CSP regressions
- Cross-site scripting
- Insecure dependency exposure
- Sensitive data leakage

## Out of Scope

- Generic support requests
- Visual defects without a security impact
- Reports without reproducible technical detail
